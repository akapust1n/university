ВАРИАНТ 04

После неправильных действий пользователя на его системном диске, ранее содержавшем операционную систему MS-DOS, осталось всего несколько файлов. При этом было обнаружено большое количество пустых каталогов и каталогов, содержащих только один подкаталог и ни одного файла. Для оптимизации структуры диска было решено пустые каталоги удалить, а если некоторый каталог X содержал только подкаталог Y, то все файлы из Y перенести в X, после чего удалить пустой теперь подкаталог Y. Эти действия надо повторять до тех пор, пока это возможно.

Требуется по списку первоначальных имён файлов с полными путями (имя дисковода не указывается) получить список имён файлов оптимизированного диска. В исходном списке содержатся только имена файлов, а не каталогов. Имена, различающиеся только регистром букв, считаются совпадающими.

Входные данные

На первой строке входного файла содержится целое число K (1 <= K <= 1000) – количество имен файлов. Следующие K строк содержат имена файлов, причем каждое имя располагается на отдельной строке без лидирующих и хвостовых пробелов. Количество символов в полном имени файла не превосходит 255.

Выходные данные

Напечатать полные имена файлов на оптимизированном диске. Имена файлов должны следовать в том же порядке, что и во входном потоке данных. Регистр букв должен быть сохранён.

Примеры

Первый пример входных данных (файл struct.in)

4
\a.001\x\a\a.doc
\A.001\x\b\columns.exe
\a.001\y
\x

Соответствующие выходные данные (файл struct.out)	

\a.001\x\a.doc
\A.001\x\columns.exe
\а.001\y
\x

Второй пример входных данных (файл struct.in)

3
\a\b\c.c
\a\B\x\c.c
\a\b\x\d

Соответствующие выходные данные (файл struct.out)

\c.c
\x\c.c
\x\d

ВАРИАНТ 23

Рассматривается каталог книг с использованием языка Simple XML (SXML), похожего на XML. Каталог содержит два типа неточностей:
1.	Некоторые книги описаны более одного раза. 
2.	Не соблюдается порядок описания элементов книги. Так, в одних описаниях книг сначала, указывается автор, а потом название, а в других - наоборот.
Требуется написать программу, в которой определяется, сколько же различных книг хранится в библиотеке.

На языке SXML описание состоит из вложенных элементов, каждый из которых состоит из открывающегося тега, вложенных элементов и закрывающегося тега:

 <имя_элемента>...вложенные_элементы...</имя_элемента>

Все SXML-описания корректны:
•	Каждое имя элемента состоит не более чем из 10 строчных букв английского алфавита.
•	Вложенность элементов не превышает 10, корневой элемент только один.
•	Строка с одним полным описанием не превосходит 250 символов.
•	Никаких других символов в описании не встречается. В частности, внутри элементов нет никаких текстовых данных. 

Пример полного SXML описания: 

<a><b></b><c><d></d></c></a>

Два описания считаются равными, если построенные деревья SXML-описаний совпадают без различения порядка следования веток у каждого из поддеревьев вершины. Элементы считаются равными, если равны их названия. Таким обра¬зом, следующие два описания считаются равными: 

<а><b></b><с></c></а> и <а><с></c><b></b></а>.

Формат входного файла

Входной файл содержит SXML-описания книг. Каждое описание размещается на отдельной строке текста. Последняя строка содержит только слово «end» (без кавычек). Количество описаний книг в одном тесте не превышает 200.

Формат выходного файла

Выходной файл содержит одно число – количество различных книг в библиотеке.

biblio.in

<a><b></b><c></c></a>
<а><c></c><b></b></а>
end

biblio.out

1

biblio.in

<a><b></b><c><d></d></c></a>
<а><b></b><с></c></а>
end

biblio.out

2
